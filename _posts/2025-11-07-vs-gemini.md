# Visual Studio - Gemini CLI 사용하기
> 참고로 미리 Google AI studio에서 API를 발급받아야 한다.

## 1. `NVM` 설치
* 관리자 권한으로 터미널 실행
* `Chocolately` 패키지 관리자 설치. ([설치 링크](https://chocolatey.org/install))

<br/>

```Powershell
Set-ExecutionPolicy Bypass -Scope Process -Force;[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
```

<br/>

* 설치 후, 새로운 터미널 창에서 `choco --version` 으로 확인
* `choco --version` 실행 시 버전이 출력된다면, `NVM` 설치

<br/>

```Powershell
choco install nvm
```

<br/>

* 설치 후, 새로운 터미널 창에서 `nvm --version` 으로 확인

## 2. `node js` 설치
* `NVM` 으로 `node js` 설치

<br/>

```powershell
nvm install lts
```

<br/>

* 설치한 `node js` 버전으로 전환
* 이 과정을 반드시 해야만 `node js` 를 사용할 수 있다.

<br/>

```powershell
nvm use lts
```

<br/>

* Windows Powershell에서는 스크립트 실행에 대해 보안 설정이 걸려 있어 `node js` 의 실행이 막혀 있다.
* 따라서, 현재 사용자에 대해 스크립트를 실행할 수 있도록 허가 해주어야한다.

<br/>

```Powershell
Get-ExecutionPolicy
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

<br/>

## 3. Gemini-CLI 설치
* `node js` 와 함께 `npm` 도 설치 되어 있다.
* 설치한 `npm` 으로 `gemini-cli` 설치

<br/>

```Powershell
npm install -g @google/gemini-cli
```

<br/>

* 설치를 완료되면 `gemini` 실행

<br/>

```Powershell
gemini
```

## 4. Visual Studio에 적용
* 작업할 프로젝트의 파일 구조는 다음과 같습니다.

<br/>

```text
Project\
├── .assistant\
│   ├── settings.json
│   └── context.md
├── src\
│   └── main.py
└── analyze.bat
```

<br/>

* `.assistant` 폴더에서 `settings.json`, `context.md` 등의 파일을 작성합니다.
* `settings.json` 은 LLM의 답변 세팅, `context.md` 는 LLM에게 전달할 맥락을 저장

<br/>

```json
// settings.json
{
  "selectedAuthType": "oauth-personal",
  // 답변의 최대 길이.
  // 최대 65536까지만 가능, 실제로는 8192만 해도 적절.
  "maxOutputTokens": 8192,
  // 무작위성의 수준.
  // 0에 가까울 수록 가장 확률이 높은 단어들만 선택, 1에 가까울 수록 무작위성이 증가.
  "temperature": 0.1,
  // 다음 단어를 정할 때의 단어 후보 범위.
  // 0에 가까울 수록 적은 후보군에서 선택, 1에 가까울 수록 넓은 후보군에서 선택
  "topP": 0.5
}
```

<br/>

```markdown
<!--context.md-->
# 이 파일의 목적
이 프롬프트는 프로젝트의 맥락을 설정하는 역할을 하는 프롬프트입니다.

# 프로젝트의 목적
이 프로젝트는 C++을 이용하여 자료구조를 공부하고 직접 구현하기 위한 프로젝트입니다.

# 역할
당신은 C++ 전문가이자 20년 이상의 경력을 가진 소프트웨어 엔지니어입니다. 당신의 역할은 코드 작성이 아닌, 코드를 분석과 리뷰를 진행하는 것입니다.
이때 고려해야할 것은 OOP, SOLID 원칙, 디자인 패턴 입니다. 그 중에서도 OOP를 가장 중요하게 생각하며, 그 다음은 SOLID 원칙, 마지막으로 디자인 패턴을 고려합니다.

코드를 전달하면, 그 코드에 맞는 피드백을 MD 파일 형식으로 생성합니다. 이때, 파일의 이름은 지금 `[YYYYMMDD]-[FileName]-feedback.md` 형식으로 생성합니다. 예를 들어, 오늘이 2025년 9월 15일이고, 피드백을 요청한 파일이름이 `main.cpp`이라면, 피드백 파일의 이름은 `20250915-main-feedback.md`가 됩니다. 만일 동일한 이름의 파일이 있는 경우에는 뒤에 숫자를 1씩 증가 시킵니다. 예를 들어 `20250915-main-feedback.md`가 존재한다면, `20250915-main-feedback1.md`이 됩니다.

피드백 파일은 다음과 같이 구성되어야 합니다.
1. 심각한 문제점: 코드에서 발견된 심각한 문제점들을 나열합니다. 코드 자체가 제대로 동작하지 않을 가능성이 있거나, 보안적 취약점이 있는 경우, 성능에 큰 영향을 미치는 경우 등이 이에 해당합니다. 각 문제점에 대해서는 구체적으로 어떤 부분이 왜 문제인지 논리적으로 설명합니다.
2. 개선할 점: 1과 같이 심각한 문제는 되지 않으나, 코드의 품질을 향상시킬 수 있는 부분들에 대해서 나열합니다. OOP 원칙, SOLID 원칙, 디자인 패턴 등이 여기에 해당됩니다. 각 문제점에 대해서는 구체적으로 어떤 부분이 왜 문제인지 논리적으로 설명합니다.
3. 참고할 점: 2와 같이 개선할 점은 아니지만, 코드 작성 시에 참고할 만한 사항들에 대해 나열합니다. 예를 들어, 코드 가독성을 위해 변수와 함수의 이름을 더 명확하게, 또 어떻게 지을 수 있는지 등에 대한 것들입니다. 각 문제점에 대해서는 구체적으로 어떤 부분이 왜 문제인지논리적으로 설명합니다.
4. 잘한 점: 코드에서 "이부분은 훌륭하다"만한 점들에 대해서 나열합니다. 어떤 부분이 구체적으로 왜 훌륭한지에 대해서도 설명합니다. 단, 이전에 언급된 심각한 문제점이나 개선할 점에 대해서 개선이 되었다면, 해당 부분의 내용만 포함합니다.

# 지침
- 피드백은 피드백 내용만 있어야 합니다. 불필요한 인사말이나 맺음말은 포함되지 않습니다.
- 피드백은 건조한 어조로 작성되어야 합니다. 당신은 감정이 없으며, 오직 논리적으로만 피드백을 작성합니다.
- 피드백은 매우 상세하게 작성되어야 합니다. 각 문제점에 대해서는 구체적으로 어떤 부분이 왜 문제인지 논리적으로 설명합니다.
- 피드백은 매우 구조적으로 작성되어야 합니다. 각 섹션은 번호로 구분되며, 각 문제점에 대해서는 번호로 구분됩니다.
- 피드백은 매우 객관적으로 작성되어야 합니다. 제안 사항 이외의 당신의 개인적인 의견이나 감정은 포함되지 않습니다.
- 피드백의 잘한 점은 이전에 언급된 심각한 문제점이나 개선할 점에 대해서 개선이 되었다면, 해당 부분의 내용만 포함합니다. 그렇다고 해서 잘한 점에 대해서 너무 과대평가하지는 않습니다. 이 역시 객관적이어야만 합니다.
```

## 5. Visual Studio 외부 도구 설정
* Visual Studio의 Powershell에서 Gemini-CLI를 일일이 켜서 사용하기에는 로딩 시간이 너무 길고, Powershell이 불안정하다.
* 때문에, `python` 으로 LLM에게 명령을 전달할 간단한 스크립트를 만들어 이를 외부 도구로 설정해서 사용한다.

<br/>

```python
## anaylze.py
import sys
import os
import subprocess

def main():
    paths = sys.argv[1:]
    
# 인자들을 받습니다.
    if not paths:
        print("[AI Assistant] 오류: 분석할 파일이나 폴더 경로를 전달해야 합니다.")
        return

    print("[AI Assistant] 전달된 파일과 폴더를 분석 중입니다...")
    
# 인자에 해당하는 파일들의 경로를 문자열로 만듭니다.
    file_contexts = []
    for path in paths:
        if os.path.isfile(path):
            print(f"  - 파일 발견: {path}")
            file_contexts.append(f'@file:"{os.path.abspath(path)}"')
        elif os.path.isdir(path):
            print(f"  - 폴더 발견: {path}")
            for root, _, files in os.walk(path):
                for file in files:
                    file_path = os.path.join(root, file)
                    file_contexts.append(f'@file:"{os.path.abspath(file_path)}"')
        else:
            print(f"  - 경고: '{path}'는 유효한 파일이나 폴더가 아닙니다.")

    file_context_string = " ".join(file_contexts)
    
    print("\n----------------------------------------------------------------------")
    print("[AI Assistant] 아래 파일들이 최종 컨텍스트에 포함되었습니다.")
    print(file_context_string)
    print("----------------------------------------------------------------------\n")
    
# 사용자의 질문을 입력합니다.
    try:
        user_prompt = input("[User] 질문을 입력하세요: ")
    except KeyboardInterrupt:
        print("\n[AI Assistant] 작업을 취소했습니다.")
        return
        
# 최종적으로 Gemini-CLI에게 넘겨줍니다.
    command = [
        "gemini",
        "--yolo",
        "@file:context.md",
        *file_contexts,
        "---",
        "질문:",
        user_prompt
    ]
    print("\n[AI Assistant] Gemini에게 작업을 요청합니다...")

    subprocess.run(command, shell=True)

if __name__ == "__main__":
    main()
```

<br/>

* `anaylze.py` 앱은 다음과 같이 사용한다.

<br/>

```Powershell
.\anaylze.py "main.cpp" "includes"
```

* 프로그램 상단의 `도구 > 외부 도구` 를 선택한다.

<br/>
<img src="/assets/image/2025-11-07-vs-gemini/3.png" width=600>
<br/>

* 여기에서 `추가` 를 선택해서 `제목` , `명령` , `인수` , `초기 디렉터리` 를 채운다.
    * 이때 python 자체 출력 창이 이미 존재하기 때문에 `출력 창 사용` 을 체크를 해제한다.
    * 만일 batch 기반이라면 체크를 해야한다.
* 각 요소들의 입력값은 다음과 같다.
    * `제목` : 원하는 데로 입력하고 `&[알파벳]` 형태로 두어 단축키를 설정한다.
    * `명령` : 실행할 프로그램을 선택한다. python의 경우 python이 설치되어 있는 경로를, batch라면 `cmd.exe`가 설치되어있는 경로로 설정
    * `인수` : 명령어의 입력값을 입력한다. 여기에서는 `"$(ProjectDir)analyze.py" "$(ItemPath)"`로 설정
    * `초기 디렉터리` : 명령어가 실행될 디렉터리를 선택한다. 여기에서는 프로젝트 디렉터리의 루트 경로인 `$(ProjectDir)`로 설정
* 이렇게 설정한 후 `도구` 에 새로운 버튼이 추가된 것을 확인할 수 있다.
