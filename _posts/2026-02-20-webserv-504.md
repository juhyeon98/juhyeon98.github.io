# 42 과제 webserv - CGI 요청 시 504 Error 발생

## 문제점 파악 1 - 504 Time out
* RFC3875를 만족하는 일반적인 CGI(FastCGI가 아님)를 구현하기 위해 `socketpair`를 사용했다.
* 기본적인 구조는 간단하다.
    1. 부모 프로세스에서 자식 프로세스와의 통신을 위한 소켓을 생성(`socketpair`)한다.
    2. 자식 프로세스를 생성(`fork`)한다.
    3. 자식 프로세스에서는 소켓의 `WRITE`(`0`) 부분을 입출력으로 바꾸고(`dup2`), 모두 닫는다.
    4. 그러고 나면, 자식 프로세스에서 CGI 환경변수와 매개변수를 설정하여 요청한 CGI를 처리한다.(`exec`)
    5. 부모 프로세스에서는 소켓의 `READ`(`1`) 만 `kqueue` 에 등록하고 나머지는 모두 닫는다.
* 이를 코드로 작성하면 아래와 같다.

<br/>

```c++
int sv[2] = { 0, };
if (socketpair(AF_UNIX, SOCK_STREAM, 0, sv) != 0) {
	throw \
		ConnectionException( \
			"Fail to create CGI file", INTERAL_SERVER_ERROR);
}
fcntl(this->mCGIfd, F_SETFL, O_NONBLOCK);

this->mCGIproc = fork();
if (this->mCGIproc < 0)
{
	throw ConnectionException( \
		"Fail to create CGI process", INTERAL_SERVER_ERROR);
}
else if (this->mCGIproc == 0)
{
	dup2(sv[WRITE], STDIN_FILENO);
	dup2(sv[WRITE], STDOUT_FILENO);
	close(sv[WRITE]);
	close(sv[READ]);
	this->addEnv(envp);
	char ** CGIenvp = this->convert(envp);
	char * argv[] = {
		const_cast<char *>(this->mCGI.c_str()),
		NULL
	};
	int ret = execve(argv[0], argv, CGIenvp);
	exit(ret);
}
close(sv[WRITE]);
this->mCGIfd = sv[READ];
kque.addCGI(this->mCGIfd, this);
this->mStatus = PROC_CGI;
gettimeofday(&this->mCGIstart, NULL);
```

<br/>

* 그리고 간단한 테스트를 위해 아래와 같은 `perl` 파일을 만들어 두었다.
* CGI 요청이 가면, 200과 함께, `"Hello, world!"` 가 출력 되어야 한다.

<br/>

```perl
print "Hello, World!\n";
```

<br/>

* 그러나 예상과는 달리 504 Time out이 되어 돌아오는 결과가 도출되었다.

## 문제점 파악 2 - 차선책으로 해결
* 이를 해결하기 위해 `pipe` 두 개 사용해 보거나 단위 테스트를 통해 문제점을 찾으려 했지만 찾지 못했다.
* 혹시나 하는 생각에 `fd` - 일반 파일을 사용하여 구현해 본 결과 아주 잘 동작했다.

<img src="/assets/image/2026-02-20-webserv-504/2.png">

* 하지만, 이 방식은 I/O 작업이 직접적으로 일어나기 때문에 그만큼 오버헤드가 많을 것으로 예상했다.
* `siege` 테스트 결과, 아래와 같이 나왔다.
* 특히나 두번 째 이미지는 `lsof` 로 모니터링한 결과인데, 여기에서 오버헤드로 인해 대기 중인 요청 - 열린 CGI 임시 파일들이 대거 있는 것을 확인할 수 있었다.

## 해결 1 - `FD_CLOSEXEC`
* 다시 문제점 1로 돌아가, 본질적인 문제가 무엇인지를 파악하고자했다.
* 예측하기로, 이는 소켓의 참조되는 count가 문제라고 생각했다.
* 즉, 자식 프로세스에서는 오로지 표준 입출력만 존재해야하는데, `kqueue`의 `fd` 를 비롯한 부모 프로세스의 수많은 소켓들이 그대로 자식 프로세스에 열린 상태로 존재하기 때문인 것이라 예상했다.
* 그래서 `fcntl` 를 통해서 CGI를 제외한 모든 소켓과 `kqueue fd` 에 `FD_CLOSEXEC` 를 설정하였다.
    * `FD_CLOSEXEC` 는 `exec` 함수가 호출되는 순간 자동으로 닫히게 한다.
* 그 결과, 더 이상 504 Time out이 나지 않고 정상적으로 동작하는 것을 확인할 수 있었다.

<img src="/assets/image/2026-02-20-webserv-504/5.png">

* 그러나 또 다른 문제점이 있었는데, 그건 `siege` 테스트 시에 일정 시간이 지나면 500 Server Error가 발생한다는 것이다.

<img src="/assets/image/2026-02-20-webserv-504/6.png">

* 이는 좀비 상태로 떠다니는 자식 프로세스의 문제였다.
* 실제로 테스트를 위해 `lsof` 를 실행하려는 순간 **`lsof` 에서 `fork` 에러가 발생** 했다.
* 이 말의 즉은, 시스템에 더 이상 생성할 프로세스가 없다는 의미였고, webserv에서 죽프로세스를 무수히 만들어내고는 회수하지 않았다는 의미이다.

<img src="/assets/image/2026-02-20-webserv-504/7.png">

## 해결 2 - `waitpid` 로 자식 프로세스 기다리기
* 현재, webserv를 실행하는 시스템은 Unix(MacOS)이다.
* Unix에서는 자식 프로세스를 생성하고나면, 부모 프로세스가 `wait` 할때 까지 `kill` 하지 않고 살려둔다.
    * 커널은 우선 자식 프로세스의 exit code를 저장하고, 자식 프로세스를 좀비 상태로 둔다.
    * 이후에 부모 프로세스에서 exit code가 필요하다면 전달해주고, 그렇지 않다면 폐기해버린다.
* 따라서 webserv(부모 프로세스)에서 **자식 프로세스가 따로 실행하고 종료할 수 있도록** 해야한다.
* 이는 다음과 같은 코드로 설정할 수 있다.

```c++
waitpid(this->mCGIproc, NULL, WNOHANG);
```

* 그 결과, 기존의 500 Server error가 사라지고 거의 모든 요청이 Hit 되는 것을 확인 할 수 있었다.
* 그리고 두번째 이미지 - `lsof` 모니터링 결과, 큰 오버헤드 없이 잘 동작하는 것을 확인 할 수 있다.

<img src="/assets/image/2026-02-20-webserv-504/8.png">
<img src="/assets/image/2026-02-20-webserv-504/9.png">

* 실제 수정된 CGI 코드는 다음과 같다.

<br/>

```c++
void Connection::processCGI( \
	Kqueue & kque, std::map<std::string, std::string> envp)
{
	int sv[2] = { 0, };
	if (socketpair(AF_UNIX, SOCK_STREAM, 0, sv) != 0) {
		throw ConnectionException( \
		"Fail to create CGI file", INTERAL_SERVER_ERROR);
	}

	fcntl(sv[WRITE], F_SETFD, FD_CLOEXEC);
	fcntl(sv[READ], F_SETFD, FD_CLOEXEC);

	int proc = fork();
	if (proc < 0) {
		throw ConnectionException( \
		"Fail to create CGI process", INTERAL_SERVER_ERROR);
	} else if (proc == 0) {
		dup2(sv[WRITE], STDOUT_FILENO);
		dup2(sv[WRITE], STDIN_FILENO);
		close(sv[WRITE]);
		close(sv[READ]);
		this->addEnv(envp);
		char **cgi_envp = this->convert(envp);
		char *argv[] = {
			const_cast<char *>(this->mCGI.c_str()),
			const_cast<char *>(this->mAbsolutePath.c_str()),
			NULL
		};
		int ret = execve(argv[0], argv, cgi_envp);
		exit(ret);
	}
	
	if (this->mRequest.getMethod() == POST) {
		std::string body = this->mRequest.getBody();
		write(sv[WRITE], body.c_str(), body.size());
	}
	close(sv[WRITE]);
	this->mCGIfd = sv[READ];
	this->mCGIproc = proc;
	fcntl(sv[READ], F_SETFL, O_NONBLOCK);
	kque.addCGI(sv[READ], this);
	this->mStatus = PROC_CGI;
	gettimeofday(&this->mCGIstart, NULL);
}

// ...

void Connection::fillRequestCGI(void)
{
	char buffer[BUFFER_SIZE];
	std::string ret;

	while (true)
	{
		int length = read(this->mCGIfd, buffer, BUFFER_SIZE - 1);
		if (length <= 0)
			break ;
		buffer[length] = '\0';
		ret += buffer;
	}
	close(this->mCGIfd);
	waitpid(this->mCGIproc, NULL, WNOHANG);
	this->mResponse.setBody(ret);
	this->mCGIfd = -1;
	this->mCGIproc = -1;
	this->mStatus = CGI_COMPLETE;
}

// ...

void Connection::closeSocket(void)
{
	if (this->mSocket != -1)
	{
		close(this->mSocket);
		this->mSocket = -1;
	}
	if (this->mCGIproc != -1)
	{
		kill(this->mCGIproc, SIGKILL);
		waitpid(this->mCGIproc, NULL, WNOHANG);
		this->mCGIproc = -1;
	}
    // ...
}
```
